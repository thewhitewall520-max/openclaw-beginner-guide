OpenClaw 是什么

一句话说清楚

OpenClaw 是一个装在你电脑上的 AI 管家系统。你告诉它做什么，它帮你干活——可以分析数据、写代码、整理文档，还能连接微信、飞书、Telegram，让你用手机就能随时发号施令。

🤔
它和直接用 ChatGPT / Claude 有什么区别？
对比项普通 AI 聊天OpenClaw
能操作你的文件吗✕ 不行✓ 可以读写文件、执行命令
手机随时控制✕ 要开网页✓ 连接飞书/Telegram 用手机发消息就行
同时用多个 AI 分工合作✕ 不行✓ 主控+专员多 Agent 团队
记得你之前说的话△ 关了对话就忘✓ 有记忆文件，跨对话保留
数据在自己电脑上✕ 全在对方服务器✓ 本地运行，数据不出门

它怎么运作的（不需要完全懂，大概知道就行）

OpenClaw 在你电脑上跑一个叫 Gateway（网关）的程序，它负责：

  • 连接各种聊天软件（Telegram、飞书等）——你发消息它就收到了
  • 把你的消息发给大模型（DeepSeek、Claude 等）——让 AI 来想怎么回应
  • 把 AI 的回应执行出来——比如真的去读文件、真的去运行一段代码
  • 把结果发回给你

你的角色：只需要发消息告诉它干什么。Gateway 和 AI 之间的通信全自动。

为什么要装虚拟机

装在哪台机器，很重要

OpenClaw 能操作你电脑上的文件和命令，权限很大。如果它出错或者被恶意利用，轻则删文件，重则影响整台电脑。

⚠️ 真实发生过的安全问题

2026年初，OpenClaw 被发现一个严重漏洞（CVE-2026-25253，危险等级 8.8/10）：攻击者只需让你点一个链接，就能在你的电脑上执行任意命令。如果 OpenClaw 装在工作机上，这意味着你的全部文件都可能被波及。

官方现已修复，但新手阶段不了解边界，用虚拟机隔离是最安全的选择。

✦ 推荐 · 本教程方案
🖥️ 虚拟机（VMware）
在你现有的电脑里建一个隔离的 Ubuntu 虚拟机，OpenClaw 装在里面。出问题最多影响虚拟机，你电脑上的真实文件安全。
○ 也可以
☁️ 云服务器
买一台阿里云/腾讯云服务器（2核4G，约 68-99 元/年）。出问题只影响云服务器，本地完全安全。但有些需要本地权限的功能用不了。
○ 最好（如果有）
🍎 闲置电脑
有闲置 Mac 或 Linux 电脑是最理想的。macOS 上 OpenClaw 功能最完整，完全不影响主力电脑。
✕ 新手不建议
💻 直接装在工作电脑
技术上可以，但 Agent 的权限和你的账号一样大。新手摸不清边界，容易出意外。熟悉之后再考虑。

Windows 电脑 → 装 VMware 虚拟机 → 虚拟机里装 Ubuntu 24.04 → Ubuntu 里装 OpenClaw。安全、可控、遇到问题容易恢复。

01 · 安装准备

开始之前先检查这些

在开始装任何东西之前，先确认你的电脑满足基本要求，然后按顺序准备好需要的软件和文件。

运行 OpenClaw 的环境要求

📋 官方系统要求（来自 docs.openclaw.ai/install）
  • Node.js 24（推荐）或 Node 22.16 以上——安装脚本会自动处理，你不用手动安装
  • 操作系统：macOS、Linux、或 Windows（WSL2 更稳定）
  • 本教程用 Ubuntu 24.04 LTS 虚拟机，满足以上所有要求

你的 Windows 电脑需要满足

项目最低要求推荐配置说明
内存（RAM）16 GB32 GB+虚拟机要分走 8GB
硬盘空闲空间70 GB100 GB+建议放在非 C 盘
处理器（CPU）4核8核+虚拟机要分走 4核
网络能上网稳定宽带下载镜像和 API 调用
怎么查看自己电脑的内存和硬盘？

按 Win + E 打开文件管理器 → 右键「此电脑」→「属性」，可以看到内存大小。硬盘剩余空间在文件管理器左侧每个盘下面都有显示。

第一步：下载并安装 VMware Workstation Pro

VMware 是用来建虚拟机的软件，现在对个人用户完全免费。

1
去 Broadcom 官网注册账号
打开浏览器，访问 broadcom.com → 右上角点「Register」→ 填写邮箱和基本信息注册 → 验证邮箱。
2
找到并下载 VMware Workstation Pro
登录后，在顶部搜索「VMware Workstation Pro」→ 点进去找「VMware Workstation Pro for Windows」→ 选最新版本下载（文件大约 600MB）。
Broadcom 网站很复杂，找不到？

直接搜索「VMware Workstation Pro 下载」，很多国内教程都有直链，或者搜索「VMware 25H2 下载」找最新版。

3
安装 VMware
双击下载的 .exe 文件 → 一路点「下一步」→ 如果问是否安装增强的键盘驱动，选「是」→ 安装完成后重启电脑（必须重启，不然虚拟机可能跑不起来）。

第二步：下载 Ubuntu 镜像文件

Ubuntu 是一种免费的 Linux 操作系统。我们要把它装进虚拟机里，再在里面装 OpenClaw。

💡 为什么选 Ubuntu 24.04 LTS？

LTS 是「长期支持」版本，稳定、有人维护、bug 少。24.04 是当前最新的 LTS，和 OpenClaw 兼容性最好。22.04 也可以用，已经有的不用重新下。

国内下载地址（速度快）
# 清华大学镜像站（推荐）
https://mirrors.tuna.tsinghua.edu.cn/ubuntu-releases/24.04/

# 找到并下载这个文件（约 5.7GB）：
ubuntu-24.04.2-desktop-amd64.iso

# 备用：阿里云镜像
https://mirrors.aliyun.com/ubuntu-releases/24.04/
下载完怎么确认文件没坏？

正常下载完的 .iso 文件大小应该在 5.5~5.8GB 左右。如果文件太小（比如只有几十 MB），说明下载中断了，要重新下。

第三步：在 VMware 里建虚拟机

1
打开 VMware，新建虚拟机
打开 VMware Workstation → 点左上角「文件」→「新建虚拟机」→ 选「典型（推荐）」→ 点「下一步」。
2
选择 Ubuntu 镜像文件
选「安装程序磁盘映像文件（iso）」→ 点「浏览」→ 找到你刚下载的 ubuntu-24.04.2-desktop-amd64.iso 文件 → VMware 会自动识别为 Ubuntu，显示「将使用简易安装」→ 点「下一步」。
找不到下载的文件？

一般在「下载」文件夹里。按 Win + E 打开文件管理器，点左边「下载」就能找到。

3
填写虚拟机信息
全名（Full name）：随便填，比如 OpenClaw
用户名（User name）：填英文，比如 claw（后面要用）
密码：设一个你记得住的密码（一定要记住！后面每次输命令都要用）
然后点「下一步」。
4
设置虚拟机名称和存放位置
虚拟机名称：填 Ubuntu-OpenClaw（方便认）
位置：点「浏览」，选非 C 盘的路径，比如 D:\VMs\Ubuntu-OpenClaw（D 盘要有至少 70GB 空间）
然后点「下一步」。
⚠️ 不要放在 C 盘！

虚拟机文件很大（装完可能占 20-40GB），放 C 盘很容易把系统盘撑满，导致电脑变慢甚至无法启动。一定要选 D 盘或其他盘。

5
设置磁盘大小
最大磁盘大小填 60（单位 GB）→ 选「将虚拟磁盘存储为单个文件」→ 点「下一步」。
6
自定义硬件（重要！不要跳过）
点「自定义硬件」→ 做这两个修改：

内存：把滑块或数字改为 8192（即 8GB）
处理器：核心数量改为 4

改完点「关闭」→ 点「完成」。
为什么要改这些？

VMware 默认只给虚拟机分配很少的内存（可能只有 2GB），OpenClaw 运行需要至少 4GB，多个 Agent 工作时需要 6-8GB。内存不够会导致系统卡顿甚至崩溃。

7
等待 Ubuntu 自动安装
VMware 会自动启动虚拟机并开始安装 Ubuntu（「简易安装」模式，全自动）。这个过程大约需要 15-30 分钟，你可以去做别的事，看到桌面出现就说明装好了。

安装完成后，虚拟机会自动重启，用你设置的用户名和密码登录。
8
安装 VMware Tools（让虚拟机更好用）
Ubuntu 启动后，在 VMware 顶部菜单点「虚拟机」→「安装 VMware Tools」。装完后，你可以在 Windows 和 Ubuntu 之间复制粘贴文字，更方便操作。

在 Ubuntu 里按 Ctrl+Alt+T 打开终端，输入：
bash
sudo apt install open-vm-tools-desktop -y
输入后按回车，它会问你密码，输入你之前设置的密码（输入时屏幕上不显示任何字符，这是正常的），输完按回车。
✓ 验证虚拟机正常

在 Ubuntu 的终端里输入 ping -c 4 baidu.com，如果看到数据在传输说明网络正常。看到「4 packets transmitted」就可以继续下一步了。

⚡ 终端是什么？怎么打开？

终端（Terminal）就是那个黑色的命令行窗口，你在里面输入命令来操作电脑。在 Ubuntu 里按 Ctrl + Alt + T 就能打开。后面所有命令都在这里输入。

02 · 安装 OpenClaw

在 Ubuntu 虚拟机里安装 OpenClaw

以下所有步骤都在 Ubuntu 虚拟机里操作。打开终端（Ctrl+Alt+T），照着输命令就行。

📋 输命令的注意事项

怎么输命令：把命令复制，在终端里点右键选「粘贴」（或者按 Ctrl+Shift+V），然后按回车执行。

输入密码时：屏幕上不会显示任何字符（不是没输进去，是故意隐藏的），输完直接按回车。

出现 [Y/n] 问你的：输入 y 然后回车，表示同意继续。

第一步：更新系统，安装 curl

bash
sudo apt update && sudo apt install -y curl

这条命令做两件事：更新软件列表，然后安装 curl（一个用来下载文件的工具）。可能需要等 1-2 分钟。

第二步：运行 OpenClaw 安装脚本

这是官方的一键安装脚本，它会自动帮你安装好 Node.js（OpenClaw 运行需要它）和 OpenClaw 本身。

bash
curl -fsSL https://openclaw.ai/install.sh | bash

这个命令会运行一段时间（可能 5-10 分钟），等它完成后，会自动进入引导配置界面。

💡 Node.js 是什么？为什么要装它？

Node.js 是 OpenClaw 运行所需要的「基础环境」——就像 Python 程序需要装 Python 才能跑，OpenClaw 需要 Node.js 才能跑。安装脚本会自动安装 Node.js 24，你不需要手动操作。

第三步：引导配置（onboard）—— 每一步怎么选

安装完成后会自动进入引导界面，用方向键 ↑↓ 选择，回车确认。

📋 引导界面每一步的选择

Quick Start or Full Setup？ → 选 Quick Start（快速开始）

模型提供商（Model provider）？ → 先选 Skip for now（稍后配置，我们下一节单独处理）

Filter providers？ → 选 All providers

Keep current or update？ → 选 Keep current

Channel（通讯平台）？ → 选 Skip for now（后面单独配）

Search settings？ → 选 Skip for now

Skills？ → 选 Skip for now

Hooks？ → 选 Skip for now

启动方式（How to start the agent）？ → 选 Web UI（网页界面，最方便）

第四步：验证安装成功

引导完成后，在终端里输入这些命令验证：

bash
# 查看版本号，能看到版本号就说明安装成功了
openclaw --version

# 查看 Gateway 是否在运行
openclaw gateway status

# 检查配置有没有问题
openclaw doctor
✓ 安装成功的标志

openclaw --version 显示版本号（比如 2026.x.x）

openclaw gateway status 显示「Runtime: running」

openclaw doctor 没有 ERROR 级别的报错

第五步：打开网页控制台

bash
openclaw dashboard

这条命令会自动在浏览器里打开控制台（地址是 http://127.0.0.1:18789）。如果浏览器自动打开了一个页面，能看到对话框，说明一切正常！

⚠️ 控制台打不开？

可能是 Gateway 没启动。在终端输入 openclaw gateway status 查看状态。如果显示 stopped，输入 openclaw gateway restart 重启。

还是不行？看「08 Bug 排查」章节。

设置国内 npm 镜像（可选，让以后更新更快）

bash
npm config set registry https://registry.npmmirror.com

这条命令把 npm（Node.js 的软件包管理器）的下载源换成国内镜像，以后更新 OpenClaw 或安装插件会快很多。

03 · 配置模型

给 OpenClaw 连接一个 AI 大脑

OpenClaw 本身只是个框架，它需要连接一个 AI 大模型才能「思考」。你需要去大模型平台注册账号、获取 API Key，然后填进 OpenClaw 里。

💡 API Key 是什么？

API Key 就像一张「通行证」，OpenClaw 拿着它去访问大模型的服务，大模型才知道是你在用、然后记账收费。API Key 是一串随机字符，格式类似 sk-abc123xyz...，要保密，不要给别人。

选哪个模型？

模型价格国内直连效果推荐程度
DeepSeek V4 Flash
deepseek/deepseek-v4-flash		极低✓ 可以快，日常任务够用★ 新手首选
DeepSeek V4 Pro
deepseek/deepseek-v4-pro		低✓ 可以强，复杂推理更准★ 进阶推荐
Claude Sonnet 4
anthropic/claude-sonnet-4-6		中等✕ 需代理极强，综合最好有代理再用
GPT-4o
openai/gpt-5.5		中等✕ 需代理强—
Gemini Flash有免费额度✕ 需代理快—
⚠️ 关于 DeepSeek 旧模型名

你可能在网上看到有人用 deepseek-chat 或 deepseek-reasoner，这些是 V3.2 时代的旧名称，现在 OpenClaw 官方文档推荐使用 deepseek/deepseek-v4-flash 和 deepseek/deepseek-v4-pro。旧名称仍然能用，但新名称是 V4 模型，更新更好。

模型名称变动很快，建议随时查官方最新列表：docs.openclaw.ai/providers/deepseek

获取 DeepSeek API Key（国内可直连，新手首选）

1
注册 DeepSeek 开放平台账号
在浏览器（Ubuntu 里或 Windows 里都行）打开 platform.deepseek.com → 用手机号或邮箱注册。
2
创建 API Key
登录后，点左边「API Keys」→ 点「创建 API Key」→ 给它起个名字（比如「openclaw」）→ 点「创建」→ 把显示出来的那串字符（sk-...开头）复制保存好，它只显示一次！
3
充值（可选，但建议充）
点「充值」→ 充 10 元人民币就够用很久了。DeepSeek V4 Flash 非常便宜，10 元可以对话几千次。也可以先不充，新注册有一些免费额度。

把模型配置到 OpenClaw

回到 Ubuntu 虚拟机，在终端里运行：

bash
openclaw onboard --auth-choice deepseek-api-key

这条命令会引导你输入 DeepSeek API Key，并自动把 deepseek/deepseek-v4-flash 设为默认模型。

📋 如果想手动配置（更灵活）

也可以运行 openclaw onboard 然后手动选择，按以下顺序选：

Gateway 位置？ → 选 local

配置模块？ → 选 Model

提供商？ → 选 Custom Provider

Base URL？ → 输入 https://api.deepseek.com

API Key？ → 粘贴你的 Key

兼容性？ → 选 OpenAI-compatible

模型 ID？ → 输入 deepseek/deepseek-v4-flash

验证模型配置成功

bash
# 重启 Gateway 让配置生效
openclaw gateway restart

# 查看已配置的模型列表
openclaw models list --provider deepseek

然后运行 openclaw dashboard 打开控制台，在聊天框输入「你好」，能收到回复就说明模型配置成功了。

deepseek-v4-flash 和 deepseek-v4-pro 有什么区别？

V4 Flash：速度快、价格低，日常聊天、简单分析、整理文档用这个，新手入门用这个。

V4 Pro：慢一些、贵一些，但推理能力强很多，复杂数学、代码分析、深度分析任务用这个。

两个都可以同时配置，在对话里用 /model deepseek/deepseek-v4-pro 临时切换。

04 · 配通讯

让 Agent 出现在你的手机上

配好通讯渠道后，你就可以用手机上的 Telegram 或飞书给 Agent 发消息，不用每次都开电脑。这是 OpenClaw 最实用的功能之一。

选哪个通讯平台？

平台国内直连配置难度适合谁
Telegram✕ 需要代理最简单有梯子的个人用户
飞书✓ 直连中等国内用户首选
钉钉✓ 直连中等企业用户
企业微信✓ 直连中等需要企业认证

方案 A：接 Telegram（配置最简单）

前提：需要能访问 Telegram 的网络环境（梯子）
1
在 Telegram 里找 BotFather 创建 Bot
打开 Telegram → 搜索 @BotFather（官方机器人，蓝色认证勾）→ 发送 /newbot

它会问你：
Bot 的显示名称：随便起，比如「我的AI助手」
Bot 的用户名：必须以 bot 结尾，比如 MyClawBot_bot

创建成功后，BotFather 会给你一串 Token，格式类似：
1234567890:ABCDEFGhijklmnopqrstuvwxyz

复制并保存这串 Token！（它只显示一次，丢了就要重新创建）
2
在 OpenClaw 里配置 Telegram
在 Ubuntu 终端里运行：
bash
openclaw onboard
然后选择：Channels → Configure/link → Telegram (Bot API) → 粘贴刚才的 Token → 回车确认
3
重启 Gateway 并测试
bash
openclaw gateway restart
在 Telegram 里找到你的 Bot，发「你好」，能收到回复就配置成功了。
⚠️ 安全提醒：设置白名单

默认情况下，任何人都可以给你的 Bot 发消息并触发 Agent。一定要设置只允许你自己的账号访问。在 ~/.openclaw/openclaw.json 里加入：

json5
{
  channels: {
    telegram: {
      allowUsernames: ["你的Telegram用户名"]
    }
  }
}

方案 B：接飞书（国内推荐）

飞书用长连接方式接入，不需要公网 IP，国内直连稳定。

1
在飞书开放平台创建自建应用
用浏览器打开 open.feishu.cn → 用飞书账号登录 → 点「开发者后台」→「创建企业自建应用」→ 填写应用名称（比如「AI助手」）→ 创建完成后，在「凭证与基础信息」页面找到并记下 App ID 和 App Secret。
2
配置事件订阅（必须选长连接）
在应用设置里找「事件订阅」→ 请求方式选「使用长连接接收事件」（不要选 Webhook！ Webhook 需要公网 IP，用长连接不需要）→ 添加事件：im.message.receive_v1（接收消息）。
3
开通权限
在「权限管理」里搜索并开通以下权限（缺一不可）：
需要开通的权限
im:message:send_as_bot    # 以机器人身份发消息
im:message                # 接收消息
im:message.group_msg      # 接收群消息（如需群聊）
4
发布应用
「版本管理与发布」→「创建版本」→「发布」→ 自建应用审核自动通过，几秒完成。
5
在 OpenClaw 里启用飞书

05 · 核心概念

这几个概念不懂会很懵

不用全部背下来，看一遍有印象，用的时候回来查就行。

工作区（Workspace）—— Agent 的「家」

OpenClaw 在你的 Ubuntu 虚拟机里有一个固定的工作目录，叫做工作区，默认位置是：

路径
~/.openclaw/workspace

所有重要的配置文件都在这里。工作区就像 Agent 的「办公室」，它的记忆、规则、个性都存在这里的文件里。

怎么打开这个文件夹？

在 Ubuntu 终端里输入 cd ~/.openclaw/workspace 然后输入 ls（小写 LS），就能看到里面所有文件。

用文件管理器（图形界面）的话：按 Ctrl+H 显示隐藏文件，然后找 .openclaw/workspace 文件夹。

工作区里有哪些文件？

这些文件 OpenClaw 每次对话都会读取，相当于 Agent 的「记忆和规则」。

AGENTS.md
操作规则——告诉 Agent 怎么工作，最重要的文件。比如：要遵守哪些规则、什么时候必须问你确认、怎么汇报结果。
SOUL.md
个性和风格——决定 Agent 说话的风格、语气、有没有个性。不写的话 Agent 会很无聊。
USER.md
你的信息——告诉 Agent 你是谁、你的偏好、怎么称呼你。
IDENTITY.md
Agent 的身份——Agent 的名字、简介。安装时自动创建。
TOOLS.md
工具说明——关于本地工具的使用说明，只是给 Agent 看的提示，不控制工具开关。
MEMORY.md
长期记忆（可选）——你想让 Agent 长期记住的事。比如你的工作方式、常用偏好等。
memory/日期.md
每日记录——每天的对话记录，自动生成。
skills/
工作技能包——你安装的各种技能（Skills）放在这个子文件夹里。

SOUL.md 怎么写

SOUL.md 是让 Agent 有「个性」的文件。不写的话 Agent 很死板，写得好会更好用。

官方建议：短而有力，不要写废话

应该写什么：说话风格、是否直接、有没有幽默感、遇到问题时的态度

不要写什么：公司规定式的废话（「请保持专业」「提供全面支持」这类），写了也没用

一个实用的 SOUL.md 模板

SOUL.md 内容（复制到文件里）
# 我的风格

- 直接给结论，不废话，不用「当然！很高兴帮你！」这种开头
- 回答要简短，能一句话说清就不说两句
- 遇到我要做蠢事，直接说，别顺着我
- 不确定的事情列出来，而不是猜着往下做
- 幽默感是允许的，但不强求

# 工作方式

- 每次只解决一个明确的问题
- 完成后说明：做了什么、结果是什么、下一步你需要做什么
- 涉及删文件、改重要配置、对外发消息，必须先告诉我并等我确认

怎么写进文件？在终端里输入：

bash
nano ~/.openclaw/workspace/SOUL.md

这会打开文本编辑器，把上面的内容粘贴进去，然后按 Ctrl+X，再按 Y，再按回车保存。

AGENTS.md 怎么写

AGENTS.md 是 Agent 的「工作规则」，比 SOUL.md 更重要，直接决定 Agent 会不会乱做事。

新手版 AGENTS.md 模板

AGENTS.md 内容
# 核心规则

以下动作必须先告诉我，等我说「可以」才能执行：
- 删除任何文件或数据
- 修改配置文件、API Key、密码、token
- 重启或停止 Gateway 服务
- 发送任何对外消息（邮件、通知、接口调用）
- 上线、部署、提交到外部平台

# 汇报格式

每次完成任务后说明：
1. 做了什么
2. 结果是什么
3. 你（用户）下一步需要做什么

不要贴大段命令输出和错误日志，只说业务结论。

# 不确定时

遇到不清楚的需求，先列出问题问我，不要猜着往下做。

写入文件的命令：

bash
nano ~/.openclaw/workspace/AGENTS.md

Context（上下文）—— 最容易懵的概念

Context 就是「这次对话里，AI 能看到多少内容」。AI 不是记忆力无限的，它每次「思考」只能处理一定量的文字。这个量叫做 Context Window（上下文窗口），超过了 AI 就看不到更早的内容了。

Context 水位示意图

越满，AI 越容易「忘事」或犯迷糊

0-60%：正常，放心用
60-80%：开始写交接记录
80%+：必须开新对话，否则 AI 会乱

怎么查看当前 Context 水位？

在对话框里直接发送这些命令（不是终端命令，是发给 Agent 的）：

在对话里发送
/status          # 快速查看当前 context 占用
/context list    # 详细查看各个文件占用了多少

Context 快满了怎么办？

1
先让 Agent 做交接总结
发消息：「总结一下目前做到哪了、还差什么、下一步要做什么，写成交接记录保存到 status.md。」
2
用 /compact 压缩历史（可选）
发 /compact，OpenClaw 会把旧的对话历史压缩成摘要，释放 context 空间。
3
开新对话继续
开一个新对话，发：「读一下 status.md，告诉我上次做到哪了，然后等我说今天做什么。」

Skills（技能包）—— 教会 Agent 特定工作方法

Skills 是一个「操作手册」文件（SKILL.md），告诉 Agent 怎么完成某类特定任务。比如「怎么写前端代码」、「怎么分析数据」、「怎么写报告」。

Skills 不是工具（不增加新能力），而是指导文件（告诉 Agent 已有能力怎么用）。

Skills 在哪里？有哪些可以安装？

官方有一个 Skills 市场叫 ClawHub（docs.openclaw.ai/clawhub），你可以在里面找到别人分享的技能包。

bash · 查看已安装的 skills
# 列出当前所有已安装的技能
openclaw skills list

怎么创建一个自己的 Skill？

在工作区的 skills/ 目录里新建一个文件夹，然后在里面创建 SKILL.md：

bash
# 创建技能目录
mkdir -p ~/.openclaw/workspace/skills/我的技能

# 创建技能文件
nano ~/.openclaw/workspace/skills/我的技能/SKILL.md

SKILL.md 里写你希望 Agent 遵循的操作步骤，比如「每次分析数据时，先做什么、再做什么、最后输出什么格式」。

06 · 第一次对话

怎么开始跟 Agent 说话

所有硬件和软件都装好了，现在教你怎么真正用起来。

第一步：让 Agent 建好团队

第一次用，发这段话给 Agent（在控制台、Telegram 或飞书里都可以）。Agent 会自动根据你的描述建好团队，写好 SOUL 和配置，不需要你手动改文件。

复制这段话，发给 Agent
你是我的主控 Agent。我需要你帮我管理以下项目：

【在这里用大白话描述你要做什么，比如：
  - 我在做电商运营，需要帮我分析竞品数据、出选品报告
  - 我要写 Python 代码，帮我找 bug、整理文档
  - 我需要整理笔记和文档，帮我总结和翻译】

请根据我的项目，建好一个合适的工作团队。规则如下：
- 删文件、改密钥、重启服务，必须先问我确认，等我同意才能动
- 最终提交、上线、发布的动作，我自己来操作
- 汇报给我结论和下一步，不要贴大段命令输出
- 不确定就先停下来问我，不要猜

建好后告诉我每个成员的名字和职责，然后等我派活。
✓ 发完确认一件事

Agent 回复后，看有没有列出团队成员名单。有就建好了。没有就说「你还没建团队，现在建」。

第二步：派第一个任务

告诉 Agent 做什么时，尽量说清楚这 4 件事：

要干嘛
这次只做这一件事。一次只说一件，多了 Agent 容易搞混。
能动哪里
允许操作哪个文件夹、哪类数据。明确范围，防止 Agent 乱动。
不能动哪
明确说禁区。没有禁区就写「无」。
怎么算完
做完要给你什么：报告、代码、截图、还是直接告诉你结果？

实际例子

分析任务
这次只做这一件事：分析我发给你的这份 Excel 表格，找出销售额最高的前 5 个产品。只读这个文件，不改任何东西。做完给我一个清单，每个产品附上具体数字。
写作任务
这次只做这一件事：把我发给你的这段文字翻译成英文，保持原意不要随意改。做完直接给我翻译结果。
代码任务
这次只做这一件事：帮我写一个 Python 脚本，读取 data.csv 文件，统计每一列的平均值。只新建文件，不改现有代码。做完给我可以直接运行的代码。

第三步：配置 status.md（让下次能接上）

每次对话结束前，让 Agent 更新 status.md，下次开新对话就能无缝接上，不用重新解释。

每次结束时发这条
今天到这里。帮我更新 status.md，写清楚：
1. 今天做了什么
2. 结果是什么
3. 明天从哪里继续
下次开新对话时发这条
读一下 status.md，告诉我上次做到哪了，然后等我说今天做什么。

每天怎么跟 Agent 一起工作

建立固定的工作节奏，Agent 就能越用越顺手。

每天开始：这样启动

1
确认 Gateway 在运行
在 Ubuntu 终端里输入，看状态是否正常：
bash

在对话里发：「读一下 status.md，告诉我上次做到哪了，然后等我说今天做什么。」
3
今天只追一个最重要的任务
不要同时让 Agent 做太多事，一次专注一件，做完再开下一件。

SOP：标准任务流程

每次给 Agent 派活，按这个流程来，出错少、结果好：

📋
标准任务卡模板
每次派活前填好这 5 项
发给 Agent 的任务格式
【项目】：这属于哪个项目
【目标】：这次只做这一件事：___
【能动的范围】：只能操作 ___，其他不要动
【禁区】：不能碰 ___（没有禁区就填「无」）

交接 SOP：Context 快满了怎么做

当你发现 Agent 开始重复说话、或者 /status 显示快满了，按这个流程交接：

1
让 Agent 写交接记录
发消息：
发给 Agent
上下文快满了，先停下来。
写一份交接记录到 status.md，包含：
1. 原本的目标是什么
2. 已经完成了什么（有哪些证据/文件）
3. 现在卡在哪里
4. 下一步要做什么，谁来做

让 Agent 读一遍 status.md 给你确认没有遗漏。

开新对话继续

多 Agent 团队怎么管

当你的任务复杂了，可以让主控 Agent 带着一个团队来做。

💡 多 Agent 的核心原则

你只跟主控说话：不要同时给多个 Agent 发指令，会乱。

主控负责拆任务：你告诉主控「做 XXX」，由主控决定哪个专员来做。

职责不重叠：每个 Agent 只做一件事，不要让两个 Agent 都去改同一个文件。

让主控建团队
我需要一个团队帮我做 [项目描述]。
请建立以下角色：
- 主控（你自己）：负责协调、拆任务、汇总结果
- 专员1：负责 [具体职责]
- 专员2：负责 [具体职责]
- 风控：负责检查所有涉及删除、修改、对外发送的操作

规则：
- 专员不能自己宣布任务完成，必须经过主控确认
- 任何「改不回来」的操作，必须先问我
建好后告诉我团队成员列表。

每天结束：收尾流程

1
更新 status.md
发：「今天到这里，更新 status.md：今天做了什么、结论是什么、明天从哪继续。」
2
重要成果单独存档
发：「把今天的 [报告/代码] 保存到 outputs/日期-任务名.md，我之后查看。」
3
验证 Gateway 还在跑
openclaw gateway status，确认还在运行，才能用手机远程发消息。
08 · Bug 排查

出问题了，按这个顺序查

遇到问题不要慌，先按官方推荐的排查顺序走一遍，80% 的问题都能找到原因。

万能排查命令（按顺序跑）

bash · 按顺序执行
# 第1步：查整体状态
openclaw status

# 第2步：查 Gateway 是否在跑
openclaw gateway status

# 第3步：实时看日志（找报错信息）
openclaw logs --follow

# 第4步：让它自己检查配置问题
openclaw doctor

# 第5步：查通讯渠道是否正常
openclaw channels status --probe

看完日志后按 Ctrl+C 退出。把看到的报错内容拍照或复制下来，下面逐一对照。

常见问题和解决方法

问题一：Gateway 启动不了

症状

openclaw gateway status 显示 stopped 或 error

1
先看是不是端口被占用
bash
openclaw logs --follow
如果看到 EADDRINUSE，说明端口 18789 被占了。重启虚拟机通常能解决。
2
强制重装 Gateway 服务
bash
openclaw gateway install --force
openclaw gateway restart
3
还不行？检查配置文件
日志里如果看到 gateway.mode 相关错误，运行：
bash
openclaw onboard --mode local

问题二：发消息没有回复

症状

在控制台或 Telegram/飞书发消息，没有任何回复，也没有报错

检查模型
运行 openclaw models list，确认有配置模型。没有的话回到「03 配置模型」重新配。
检查 API Key
运行 openclaw doctor，看有没有 API Key 相关的报错。Key 过期或没余额都会导致无回复。
看实时日志
运行 openclaw logs --follow，然后发一条消息，看日志里有没有错误信息。
通讯渠道问题
运行 openclaw channels status --probe，检查 Telegram/飞书通道是否连接正常。

问题三：控制台（Dashboard）打不开

1
确认 Gateway 在运行
bash
openclaw gateway status
没在跑？运行 openclaw gateway restart
2
手动在浏览器里输入地址
不要依赖自动打开，直接在 Ubuntu 的浏览器地址栏里输入：http://127.0.0.1:18789
3
如果要求输入 Token
运行这条命令获取 Token，把输出的链接复制到浏览器：
bash
openclaw dashboard

问题四：升级后突然出问题

症状

更新了 OpenClaw 之后，Gateway 无法启动，日志里有版本不匹配相关错误

bash
# 检查版本是否一致
openclaw --version
openclaw gateway status --deep

# 重装 Gateway 服务
openclaw gateway install --force
openclaw gateway restart

问题五：Anthropic API 报 429 错误

症状

日志里看到 HTTP 429: Extra usage is required for long context requests

这是因为你的 Claude 账号没有开通长上下文权限，或者当前账号不支持。解决方案：

  • 改用 DeepSeek 模型（国内直连，没有这个限制）
  • 换有长上下文权限的 Anthropic API Key
  • 缩短对话长度（开新对话继续）

09 · 控制 Agent

怎么让 Agent 听你的话

掌握这些对话技巧，你就能真正控制 Agent，而不是被它带着跑。

纠偏：它跑偏了怎么办

它废话太多
直接给我结论，不要解释过程。
它做了不该做的
停。我没让你做这个，撤回，重新等我指令。
它问太多问题不干活
先按你现在理解的做，做完了我再纠正。
它没理解我的意思
不对，我的意思是：【重新说一遍】。这次只做这一件事。
它做到一半卡住了
现在做到哪了？还差什么？用三句话告诉我。
报告太长看不下去
重新出一版，只保留最关键的三点，其他删掉。
它开始重复犯迷糊
Context 快满了，先写交接记录到 status.md，然后我开新对话继续。
你想换个方向
停止现在的任务。我们改做：【新任务描述】。
它做完了确认结果
做完了吗？最终结果是什么？我下一步需要做什么？
它同一问题失败两次
停下来。告诉我卡在哪、缺什么，我来决定怎么处理，不要第三次重试。

危险操作黑名单

这些操作 Agent 不能自己做，必须先告诉你、等你确认。

🚫 以下操作必须先问你
  • 重启或停止 Gateway 服务 —— openclaw gateway restart/stop
  • 删除任何文件或数据 —— rm 命令、清空目录、删数据库记录
  • 修改配置或密钥 —— openclaw.json、API Key、token、环境变量
  • 对外发送消息 —— 发邮件、推送通知、调用任何对外接口
  • 上线、提交、发布 —— 部署代码、推送主分支、提交到外部平台
  • 安装或卸载系统级软件 —— sudo apt install、系统设置修改

对应的叫停句

它想重启服务
停。告诉我重启有什么后果、有没有必要。等我说可以再动。
它想删文件
停。把要删的文件列出来，我一个个确认之后再删。
它想改密钥或配置
停。告诉我改哪里、改成什么，我自己来动手。
它想替你提交或上线
停。提交这一步我自己来，你只需要把最终内容给我。
🔒 万能叫停句，记住这一句

任何时候感觉它要干危险的事，发这句话：

「停。先告诉我你要做什么、有什么后果，等我说可以再动。」
